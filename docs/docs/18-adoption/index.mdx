# Adoption Signals

> **Status**: Informative
> **Authority**: Documentation Governance

This section explains how MPLP adoption is tracked and communicated.

---

## What This Section Provides

- **Signal Definition**: What constitutes an "adoption signal"
- **Verification Method**: How signals are validated
- **Transparency**: Where signals are published

---

## What This Section Does NOT Provide

> [!IMPORTANT]
> MPLP does **not**:
> - Endorse or recommend specific implementations
> - Guarantee quality or correctness of third-party adoptions
> - Provide commercial partnership programs

---

## Adoption Signal Types

| Signal Type | Description | Verification |
|:------------|:------------|:-------------|
| **Self-Declared** | Implementer claims conformance | Public documentation |
| **Test-Verified** | Passes Golden Flow tests | Test results |
| **Community-Reviewed** | Reviewed by community | Public review |

---

## How to Declare Adoption

1. Implement MPLP according to the specification
2. Run Golden Flow tests
3. Document results publicly
4. (Optional) Submit to community registry
