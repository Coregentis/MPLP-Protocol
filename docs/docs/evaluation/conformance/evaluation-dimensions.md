# Evaluation Dimensions

This page is a **docs-side orientation page** for evaluation questions. It does
not create a frozen dimension taxonomy or an authoritative evaluator contract.

## 1. Purpose

Use this page to navigate common evaluation questions such as:

- schema validity
- evidence completeness
- version compatibility
- ruleset-based adjudication context

## 2. What This Page Can Safely Do

This page can safely point readers toward:

- repository-backed schemas and invariants for protocol-side validity
- repaired Validation Lab reference pages for ruleset/evidence-pack context
- downstream results pages for reporting terminology

## 3. What This Page Does Not Define

This page does not define:

- a frozen 6-dimension constitution
- pass/fail ownership for every evaluation question
- a canonical dimension-to-class matrix
- future version evaluation doctrine

## 4. Useful Evaluation Questions

The following questions remain useful as reading prompts, not as a separate
constitution:

| Question | Read First |
|:---|:---|
| Do artifacts match repository-backed schemas? | repository-backed schemas and repaired module/profile pages |
| Is the evidence-pack/ruleset relationship understood? | [Evidence Pack Contract](/docs/evaluation/validation-lab/evidence-pack-contract.mdx), [Ruleset Reference](/docs/evaluation/validation-lab/rulesets.mdx) |
| Is protocol versioning being read in the correct domain? | [Versioning Policy](../governance/versioning-policy.md), [Entry Points](/docs/reference/entrypoints) |
| Are adjudication targets understood? | [Lifecycle Guarantees Reference](/docs/evaluation/validation-lab/lifecycle-guarantees.mdx) |

## 5. Reading Order

1. repaired spec/reference pages for the protocol object or profile in question
2. repaired Validation Lab reference pages for evidence/ruleset context
3. [Results & Status](./results-and-status.md) for reporting terminology
