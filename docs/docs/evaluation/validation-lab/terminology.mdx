## Why Terminology Matters

MPLP uses distinct naming conventions across its four entry points. Conflating terms leads to confusion about what is being tested, evaluated, or adjudicated. This page provides the authoritative partition.

## Core Terminology Partition

:::warning Critical: Flow ≠ LG
- **Flow-01~05** = Protocol Test Scenarios (main repository)
- **LG-01~05** = Lifecycle Guarantees (Validation Lab adjudication targets)

These are **distinct naming spaces** and should not be conflated.
:::

| Term | Definition | Owned By | Used In |
|:---|:---|:---|:---|
| **Flow-01~05** | Protocol test scenarios | Main repository | `tests/golden/flows/` |
| **LG-01~05** | Lifecycle Guarantees (adjudication targets) | Validation Lab | Lab UI, Docs Reference |
| **gf-01~05** | Internal IDs for LG (frozen, do not use externally) | Validation Lab | Internal code only |
| **Scenario** | A test fixture / evidence generation context | Main repository | Test files |
| **Ruleset** | Versioned evaluation rules | Validation Lab | `data/rulesets/` |
| **Evidence Pack** | Adjudication input bundle | Validation Lab | `pack-v*` format |

## Display Rules

| Context | Use | Do NOT Use |
|:---|:---|:---|
| **External display (UI, Docs, Website)** | `LG-01`, `LG-02`, etc. | `gf-01`, `Golden Flow` |
| **Internal code (Lab only)** | `gf-01`, `gf-02`, etc. | — |
| **Protocol test scenarios** | `Flow-01`, `FLOW-01`, etc. | `LG-01` |

## Prohibited Terminology

The following terms are **forbidden** in external-facing content:

| Forbidden | Reason | Alternative |
|:---|:---|:---|
| **Golden Flow** (as Lab term) | Causes collision with main repo Flow | **Lifecycle Guarantee (LG)** |
| **gf-xx** (in UI/Docs) | Reserved for internal ID only | **LG-xx** |
| **Certified** | Lab does not certify | **Evaluated**, **Adjudicated** |
| **Badge** | Lab does not issue badges | **Verdict** |
| **Ranking** | Lab does not rank | **Coverage**, **Status** |
