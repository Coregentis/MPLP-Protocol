## Choose Your Evaluation Path

### If You Are an Investor / Board Member

**Goal**: Understand governance, maturity, and trust model

**Reading Order** (15 minutes):
1. [External Trust Overview](/docs/governance/EXTERNAL_TRUST_OVERVIEW) — What MPLP is/isn't
2. [Governance Layers](/docs/governance/GOVERNANCE_LAYERS) — How it's governed
3. [Governance Statement](/docs/governance/GOVERNANCE_STATEMENT) — External summary
4. [Standards Overview](/docs/standards/positioning) — ISO/NIST positioning

**Key Questions Answered**:
- Who controls the protocol?
- How are changes made?
- What claims are NOT being made?

---

### If You Are an Enterprise Architect

**Goal**: Assess technical fit and compliance posture

**Reading Order** (30 minutes):
1. [Architecture Overview](/docs/architecture/architecture-overview) — L1–L4 layer model
2. [Modules](/docs/modules/core-module) — 10 module specifications
3. [Golden Flows](/docs/golden-flows) — Validation scenarios
4. [Standards Mapping](/docs/standards/positioning) — ISO/NIST/W3C alignment
5. [Enterprise Evaluation](/docs/governance/agentos-protocol) — Assessment checklist

**Key Questions Answered**:
- What does the protocol define?
- How do I verify implementation correctness?
- How does this relate to our compliance requirements?

### If You Are a Contributor / Developer

### If You Are an Auditor / Compliance Reviewer

**Goal**: Gather evidence and verify claims

**Reading Order** (60 minutes):
1. [Repo-Docs-Code Alignment](/docs/index/REPO_DOCS_CODE_ALIGNMENT) — Traceability
2. [Golden Flows](/docs/golden-flows) — Validation evidence
3. [Standards Mapping](/docs/standards/positioning) — External alignment
4. [External Consistency Summary](/docs/governance/EXTERNAL_TRUST_OVERVIEW) — Controls
5. [Phase G Verification](/docs/governance/GOVERNANCE_STATEMENT) — Surface consistency

**Key Questions Answered**:
- Are claims backed by artifacts?
- Is behavior testable?
- Are governance controls documented?

## Common Misinterpretations

> [!WARNING]
> **Explicitly Disallowed Interpretations**

| ❌ Do NOT Interpret | ✅ Correct Interpretation |
|:--------------------|:--------------------------|
| "Docs is the product website" | Docs is specification, not marketing |
| "Standards mapping = compliance" | Mapping = enablement, not certification |
| "SDK is the protocol" | SDK is reference implementation |
| "MPLP certifies implementations" | MPLP defines standards, not certifications |
| "Ecosystem = endorsement" | Ecosystem is informational only |

## Verification Checklist

Before concluding your evaluation, verify:

- [ ] You understand the three-entry model (Website ↔ Docs ↔ Repo)
- [ ] You have read the Non-Claims section
- [ ] You understand who holds governance authority
- [ ] You know how to verify implementation correctness (Golden Flows)
- [ ] You understand the standards mapping scope and limitations
