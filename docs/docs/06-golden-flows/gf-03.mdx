# GF-03: Drift Detection & Recovery

> **Type**: Normative Validation Scenario
> **Profile**: SA (Single Agent)
> **Status**: ✅ VERIFIED

## Scope

This golden flow validates:
- **Drift detection** when physical files are modified outside the protocol
- **Trace** emission of `DriftDetected` events
- **Recovery Plan** generation to realign state
- **Extension** module usage for external state verification

## Non-Goals

This flow does NOT validate:
- Multi-agent coordination (see GF-02)
- Initial plan creation (see GF-01)

## Normative Requirements

Implementations MUST:
- Emit a `DriftDetected` event immediately upon discovery of divergence
- Include the `drift_scope` and `impact_analysis` in the event payload
- Generate a new `Plan` (Recovery Plan) to address the drift
- Block further execution of the original Plan until drift is resolved or acknowledged

---

## Expected Outcome

- System detects the external change (e.g., file deletion)
- `DriftDetected` event is present in the Trace
- A recovery plan is proposed and successfully executed
- Final state is consistent with the realigned reality
