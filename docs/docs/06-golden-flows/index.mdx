## What Are Golden Flows?

Golden Flows are **end-to-end execution scenarios** that:

1. **Define** the expected sequence of module interactions
2. **Validate** that implementations produce correct state transitions
3. **Provide** evidence of conformance for auditors

They are **not tutorials** — they are **contracts**.

---

## The Five Golden Flows

| Flow | Name | Purpose | Modules Involved |
|:-----|:-----|:--------|:-----------------|
| **GF-01** | [SA Lifecycle](/docs/golden-flows/gf-01) | Single-agent lifecycle correctness | Context, Plan, Trace |
| **GF-02** | [MAP Coordination](/docs/golden-flows/gf-02) | Multi-agent coordination | Collab, Role, Dialog |
| **GF-03** | [Drift Detection](/docs/golden-flows/gf-03) | Plan-execution divergence | Plan, Trace, Confirm |
| **GF-04** | [Delta-Intent](/docs/golden-flows/gf-04) | Incremental state update | Context, Trace |
| **GF-05** | [Governance](/docs/golden-flows/gf-05) | Policy enforcement | Confirm, Role |

## How to Use Golden Flows

### For Implementers
Run the Golden Flow test suite against your implementation to verify conformance.

```bash
npm run test:golden
```

### For Auditors
Use Golden Flows as **evidence** that an implementation behaves correctly.
Each flow maps to specific modules and cross-cutting duties.

### For Evaluators
Golden Flows demonstrate that MPLP is **verifiable**, not just aspirational.
