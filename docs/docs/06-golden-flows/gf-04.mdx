# GF-04: Delta-Intent (LLM Enrichment)

> **Type**: Normative Validation Scenario
> **Profile**: SA (Single Agent)
> **Status**: ✅ VERIFIED

## Scope

This golden flow validates:
- **Incremental updates** to the Project Context (Delta-Intent)
- **LLM enrichment** of raw user requests into structured objectives
- **Core** module's role as the protocol version anchor
- **Trace** recording of reasoning steps

## Non-Goals

This flow does NOT validate:
- Multi-agent coordination (see GF-02)
- Governance confirmation (see GF-05)

---

## Modules Involved

| Module | Role in Flow |
|:-------|:-------------|
| **Context** | Receives the delta updates |
| **Core** | Manages protocol versioning and module state |
| **Plan** | Updates based on enriched intent |
| **Trace** | Records the reasoning graph and intent resolution |

## Cross-Cutting Duties Exercised

- Intent Resolution
- Reasoning Trace

## Normative Requirements

Implementations MUST:
- Support `DeltaIntent` events that modify the active Context
- Record the `reasoning_graph` in the Trace for all LLM-driven enrichments
- Ensure that Delta updates do not violate existing L1 invariants
- Maintain a clear link between the original Intent and the Enriched Objective

---

## Expected Outcome

- Raw intent is successfully transformed into a detailed Plan
- Delta updates are applied without breaking context integrity
- Reasoning steps are fully auditable in the Trace
