# Adoption


## 1. What Adoption Means

Adoption is NOT:
- A certification
- A badge or endorsement
- A binary "installed / not installed" state

Adoption IS:
- A **lifecycle** with distinct stages
- A **signal-based** recognition
- A **self-declared** state (no external authority)

## 2. Adoption Lifecycle

MPLP adoption follows a three-stage lifecycle:

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  EVALUATION │ ──► │    PILOT    │ ──► │ PRODUCTION  │
└─────────────┘     └─────────────┘     └─────────────┘
```

### 2.1 Evaluation Stage

| Aspect | Description |
|:---|:---|
| **Goal** | Understand if MPLP fits the use case |
| **Activities** | Read docs, run Golden Flows, validate schemas |
| **Signals** | SDK installed, test fixtures run |
| **Evidence** | None required (no production data) |

### 2.2 Pilot Stage

| Aspect | Description |
|:---|:---|
| **Goal** | Validate MPLP in a controlled environment |
| **Activities** | Integration with real agent, limited scope |
| **Signals** | Evidence Pack generated, Conformance evaluated |
| **Evidence** | Pilot-scoped Context, Plan, Trace |

### 2.3 Production Stage

| Aspect | Description |
|:---|:---|
| **Goal** | Full operational use of MPLP |
| **Activities** | Agent system in production with MPLP governance |
| **Signals** | Ongoing evidence production, audit readiness |
| **Evidence** | Full production Evidence Packs |

## 4. Adoption Signals

See [Adoption Signals](./adoption-signals.md) for the signal definitions.

Key signals include:
- **SDK Installation** — First contact
- **Golden Flow Execution** — Technical validation
- **Evidence Pack Generation** — Pilot readiness
- **Conformance Evaluation** — Production readiness
