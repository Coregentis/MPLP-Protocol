# Conformance Checklist

Use this checklist to confirm that you are reading MPLP conformance material in
the correct order.

- [ ] I started with [Entry Points](/docs/reference/entrypoints)
- [ ] I know repository-backed schemas and invariants remain protocol truth
- [ ] I know Golden Flows are protocol-side evaluation scenarios
- [ ] I know Validation Lab is an auxiliary adjudication surface, not a
      protocol-defining primary
- [ ] I know Flow names and Lab guarantee/ruleset names are distinct
- [ ] I know `protocol_version`, `validation_ruleset_version`, and package
      versions are separate version domains
- [ ] I know package guides and helper pages are informative and subordinate to
      repaired primary docs/reference surfaces

This checklist is a reading aid only, not a certification or self-attestation
instrument.
