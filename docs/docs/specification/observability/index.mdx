# Observability Overview

> [!NOTE]
> **Non-Normative Document**
>
> This page is informative only. It restates schema structure for navigation purposes.

> **SSOT Reference**: Authoritative observability definitions live in [Repository schemas](https://github.com/Coregentis/MPLP-Protocol/tree/main/schemas/v2/events).

## What This Section Contains

MPLP specifies **observability contracts** for event and trace visibility.
This section routes readers through the event, invariant, and trace references
without creating a second observability doctrine layer.

| Document | Purpose |
|----------|---------|
| [Event Taxonomy](./event-taxonomy.md) | Event family classification |
| [Common Schemas Reference](./common-schemas-reference.md) | Shared schema elements |
| [Physical Schemas Reference](./physical-schemas-reference.md) | Storage format schemas |
| [Observability Invariants](./observability-invariants.md) | Correctness constraints |
| [Module-Event Matrix](./module-event-matrix.md) | Module to event mapping |
| [Runtime Trace Format](./runtime-trace-format.md) | Trace and export reference |
| [Observability Overview](./observability-overview.md) | High-level orientation |

## Reading Path

1. **Start with**: [Event Taxonomy](./event-taxonomy.md)
   Establish the event families first.
2. **Then**: [Common Schemas Reference](./common-schemas-reference.md) and
   [Physical Schemas Reference](./physical-schemas-reference.md)
   Ground the event family model in the underlying schemas.
3. **Then**: [Observability Invariants](./observability-invariants.md)
   Read the observability correctness layer.
4. **Then**: [Module-Event Matrix](./module-event-matrix.md)
   Read module-to-event routing after the family and schema baseline is clear.
5. **Last**: [Runtime Trace Format](./runtime-trace-format.md) and
   [Observability Overview](./observability-overview.md)
   Use these as downstream trace/export and orientation surfaces.
