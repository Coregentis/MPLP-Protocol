# Profiles Overview

> [!NOTE]
> **Non-Normative Document**
>
> This page is informative only. It restates schema structure for navigation purposes.

> **SSOT Reference**: Authoritative profile definitions live in [Repository schemas](https://github.com/Coregentis/MPLP-Protocol/tree/main/schemas/v2/profiles).

## What This Section Contains

MPLP defines **profile baselines** that constrain how protocol object families
are used in single-agent and multi-agent settings.

| Profile | Schema File | Description |
|---------|-------------|-------------|
| [Single-Agent (SA)](./sa-profile.md) | `sa-profile.yaml` | Base profile baseline |
| [Multi-Agent Protocol (MAP)](./map-profile.md) | `map-profile.yaml` | Multi-agent extension baseline |

## Profile-Linked Event References

Profile pages are linked to profile-specific event surfaces:

- [SA Events](./sa-events.md) — Single-Agent event taxonomy
- [MAP Events](./map-events.md) — Multi-Agent event taxonomy

## Secondary Governance-Oriented Reference

- [Multi-Agent Governance](./multi-agent-governance-profile.md)
  Read this after SA and MAP profile baselines. It is not the first place to
  establish profile identity.

## Reading Path

1. **Start with**: [SA Profile](./sa-profile.md)
   Establish the base profile first.
2. **Then**: [MAP Profile](./map-profile.md)
   Read MAP as an extension of SA.
3. **Then**: [SA Events](./sa-events.md) and [MAP Events](./map-events.md)
   Read profile-linked event surfaces after the base profile pages.
4. **Last**: [Multi-Agent Governance](./multi-agent-governance-profile.md)
   Use only after the base profile surfaces are clear.
