# Protocol Specification

This section provides the canonical reading path into MPLP specification
surfaces.

It is a navigational page, not an additional doctrine layer. Repository-backed
schemas, invariants, profile manifests, and approved governance records remain
the upstream truth source.

## Canonical-First Reading Order

Read the specification in this order:

1. **Modules**
   Establish the protocol object families first.
2. **Profiles**
   Establish SA/MAP profile baselines after object families are clear.
3. **Observability**
   Establish event families, observability invariants, and trace/export
   references after modules and profiles.
4. **Architecture**
   Use architecture pages to understand how the formal families relate across
   layers.
5. **Integration**
   Read L4 event and integration surfaces after the core baseline is clear.
