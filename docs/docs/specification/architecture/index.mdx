# Architecture

Welcome to the MPLP Architecture documentation. This section covers the foundational design of the protocol.

## Scope

This index provides navigation to MPLP's **four-layer architecture** and related
structural reference pages.

Architecture pages explain how protocol families relate across layers. They are
not the first place to establish formal object, profile, or event meaning.

## Non-Goals

This page does not contain normative requirements. All normative content is defined in the linked specification documents.

---

## Canonical Reading Position

For canonical-first reading:

1. Read [Modules](/docs/specification/modules),
   [Profiles](/docs/specification/profiles), and
   [Observability](/docs/specification/observability) first.
2. Use Architecture second, to understand how those formal families relate
   across L1-L4.
3. Do not read runtime-layer concepts as new protocol core objects.
