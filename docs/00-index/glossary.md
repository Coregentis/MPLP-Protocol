# Glossary

This document provides normative definitions for key terms used throughout the Multi-Agent Lifecycle Protocol (MPLP) specification.

## L1 Core Terms

### Agent
**Category:** L1 Core Protocol
**Definition (Normative):** An autonomous entity capable of perceiving its environment, reasoning about goals, and executing actions within the MPLP framework.
**Source Documents:**
- `docs/01-architecture/l1-core-protocol.md`

### Confirm
**Category:** L1 Core Protocol
**Definition (Normative):** A core module responsible for validation, approval, and risk control of planned actions before execution.
**Source Documents:**
- `docs/02-modules/module-confirm.md`
- `schemas/v2/mplp-confirm.schema.json`

### Context
**Category:** L1 Core Protocol
**Definition (Normative):** The shared state and environment information accessible to agents, serving as the grounding for all planning and execution.
**Source Documents:**
- `docs/02-modules/module-context.md`
- `schemas/v2/mplp-context.schema.json`

### MPLP (Multi-Agent Lifecycle Protocol)
**Category:** L1 Core Protocol
**Definition (Normative):** A vendor-neutral, open protocol for governing multi-agent AI systems across their full lifecycle, from planning to execution and observability.
**Source Documents:**
- `docs/00-index/mplp-v1.0-protocol-overview.md`

### Plan
**Category:** L1 Core Protocol
**Definition (Normative):** A structured sequence of proposed steps or actions generated by an agent to achieve a specific goal.
**Source Documents:**
- `docs/02-modules/module-plan.md`
- `schemas/v2/mplp-plan.schema.json`

### Trace
**Category:** L1 Core Protocol
**Definition (Normative):** An immutable record of all events, decisions, and state changes that occur during the lifecycle of an agent session.
**Source Documents:**
- `docs/02-modules/module-trace.md`
- `schemas/v2/mplp-trace.schema.json`

## L2 Module Terms

### Collab
**Category:** L2 Coordination & Governance
**Definition (Normative):** A module that manages interactions and coordination patterns between multiple agents.
**Source Documents:**
- `docs/02-modules/module-collab.md`
- `schemas/v2/mplp-collab.schema.json`

### Core (Module)
**Category:** L2 Coordination & Governance
**Definition (Normative):** A module providing fundamental system capabilities and utilities required by other modules.
**Source Documents:**
- `docs/02-modules/module-core.md`
- `schemas/v2/mplp-core.schema.json`

### Dialog
**Category:** L2 Coordination & Governance
**Definition (Normative):** A module that structures and manages conversational exchanges between agents or between agents and humans.
**Source Documents:**
- `docs/02-modules/module-dialog.md`
- `schemas/v2/mplp-dialog.schema.json`

### Extension
**Category:** L2 Coordination & Governance
**Definition (Normative):** A module that allows for the integration of custom capabilities, tools, or third-party services.
**Source Documents:**
- `docs/02-modules/module-extension.md`
- `schemas/v2/mplp-extension.schema.json`

### Network
**Category:** L2 Coordination & Governance
**Definition (Normative):** A module handling communication protocols and transport mechanisms between distributed agents.
**Source Documents:**
- `docs/02-modules/module-network.md`
- `schemas/v2/mplp-network.schema.json`

### Role
**Category:** L2 Coordination & Governance
**Definition (Normative):** A module that defines the specific responsibilities, permissions, and capabilities of an agent within a session.
**Source Documents:**
- `docs/02-modules/module-role.md`
- `schemas/v2/mplp-role.schema.json`

## L2.5 Observability Terms

### Event Bus
**Category:** L2.5 Observability
**Definition (Normative):** The mechanism for dispatching and routing events between modules and the observability layer.
**Source Documents:**
- `docs/04-observability/mplp-observability-overview.md`

### Telemetry
**Category:** L2.5 Observability
**Definition (Informational):** The collection and transmission of metrics and logs regarding the system's performance and health.
**Source Documents:**
- `docs/04-observability/mplp-telemetry-standard.md`

## L3 Runtime Terms

### Action Execution Layer (AEL)
**Category:** L3 Runtime
**Definition (Normative):** The runtime component responsible for actually performing the actions defined in a plan, often interfacing with LLMs or external tools.
**Source Documents:**
- `docs/06-runtime/mplp-runtime-glue-overview.md`

### Drift Detection
**Category:** L3 Runtime
**Definition (Normative):** The process of monitoring for and identifying discrepancies between the planned state/actions and the actual runtime execution.
**Source Documents:**
- `docs/06-runtime/drift-detection-spec.md`

### Project Semantic Graph (PSG)
**Category:** L3 Runtime
**Definition (Normative):** A unified graph representation of the project's state, including code, documentation, and agent knowledge, maintained by the runtime.
**Source Documents:**
- `docs/06-runtime/module-psg-paths.md`

### Verification & Simulation Layer (VSL)
**Category:** L3 Runtime
**Definition (Normative):** A runtime layer that validates plans and simulates outcomes before execution to ensure safety and correctness.
**Source Documents:**
- `docs/06-runtime/mplp-runtime-glue-overview.md`

## L4 Integration Terms

### Adapter
**Category:** L4 Integration
**Definition (Informational):** A component that translates between MPLP protocols and external system APIs or formats.
**Source Documents:**
- `docs/07-integration/integration-patterns.md`

### SA Profile (Single-Agent Profile)

**Definition (Normative):** A standard configuration profile optimized for single-agent autonomous tasks.
