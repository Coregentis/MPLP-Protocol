# MPLP MPGC Intake Evidence Vocabulary Candidate v0.1

## Candidate Theme

Evidence vocabulary candidate for how implementation substrates may map evidence
insufficiency, stale context, clarification, non-executing posture, and
revision references into MPLP-aligned evidence records without changing
protocol schemas.

## Possible Future Binding Consideration

- relation wording may later need a consistent non-schema mapping note
- reference-linking questions may eventually need bounded cross-implementation
  guidance
- current pressure is still pre-RFC and non-normative

## Open Mapping Questions

- how evidence insufficiency relation should be expressed without a new schema
  object
- how stale context relation should remain distinct from ordinary trace context
- how prior and revised artifact references should be linked
- how non-executing posture references should be carried without implying
  execution outcomes
- how blocked / contract-blocked relations should remain non-approval and
  non-rejection

## Candidate Vocabulary Questions

- what vocabulary best separates insufficiency from stale context
- what vocabulary best separates clarification from escalation
- what vocabulary best separates revision from approval and execution
- what vocabulary best preserves bounded safe evidence references

## Schema Risk

- premature vocabulary hardening could be mistaken for schema expansion
- relation questions could be over-read as field commitments

## Compatibility Risk

- inconsistent mapping language could fragment implementation substrates
- overly strong vocabulary could imply acceptance or compatibility guarantees

## Explicit Non-Decisions

This is not a binding spec draft.
This is not a binding change accepted by MPGC.
This does not add vocabulary to MPLP schemas.
- pre-RFC only
- non-normative only
- not protocol law

- no runtime endorsement
- no product endorsement
- no implementation endorsement
- no certification claim
