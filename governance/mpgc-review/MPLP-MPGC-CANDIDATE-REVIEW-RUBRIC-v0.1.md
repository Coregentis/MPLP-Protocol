# MPLP MPGC Candidate Review Rubric v0.1

## A. Purpose

Define review criteria for MPGC candidate discussion.

## B. Rubric Matrix

| Criterion | Question | Scoring posture | Blocking concern |
| --- | --- | --- | --- |
| protocol neutrality | does the candidate remain protocol-neutral and not product-shaped | `low / medium / high / unknown` | hidden product semantics |
| cross-runtime generality | is there evidence beyond one implementation substrate or downstream line | `low / medium / high / unknown` | overgeneralization from narrow evidence |
| lifecycle relevance | is the candidate actually about lifecycle posture rather than application convenience | `low / medium / high / unknown` | candidate theme drift |
| evidence sufficiency | is the evidence strong enough for further work-item preparation | `low / medium / high / unknown` | evidence overclaim |
| schema impact | would the candidate pressure immediately imply schema discussion | `low / medium / high / unknown` | schema creep |
| protocol law impact | would the candidate pressure immediately imply protocol-law widening | `low / medium / high / unknown` | accidental promotion |
| conformance risk | could the candidate be misread as a conformance obligation | `low / medium / high / unknown` | conformance confusion |
| endorsement risk | could the candidate be misread as endorsing a runtime or product | `low / medium / high / unknown` | endorsement leakage |
| implementation dependency risk | is the candidate still too coupled to one implementation substrate | `low / medium / high / unknown` | implementation dependence |
| adoption clarity | is it clear what future path, if any, is being discussed | `low / medium / high / unknown` | ambiguity in next-step expectations |
| backward compatibility | could future work on the candidate threaten existing interpretation surfaces | `low / medium / high / unknown` | compatibility confusion |
| documentation value | would future explanatory material add clarity without overpromoting the candidate | `low / medium / high / unknown` | documentation drift into acceptance language |

No score is assigned in this wave.
